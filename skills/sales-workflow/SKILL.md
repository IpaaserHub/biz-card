# Sales Workflow（営業支援ワークフロー）

営業活動の事前準備から事後振り返りまでを一貫してサポートするスキル。

## ワークフロー概要

| フェーズ | 目的 | テンプレート |
|---------|------|-------------|
| 1. リサーチ | 営業先の人物・企業情報を収集し仮説を構築 | `templates/research_template.md` |
| 2. 台本作成 | リサーチ結果をもとに商談の流れを設計 | `templates/script_template.md` |
| 3. フィードバック | 議事録を分析し改善点と次回アクションを提案 | `templates/feedback_template.md` |

## 実行手順

### フェーズ1: 営業先リサーチ

ユーザーから営業先の人物名・企業名を受け取り、以下を実行する。

1. `search` ツールで企業情報を収集（事業内容、直近ニュース、業界動向）
2. `search` ツールで人物情報を収集（経歴、SNS発信、インタビュー記事、登壇実績）
3. 収集した情報をもとに仮説を構築（推測される課題、自社サービスが提供できる価値）
4. `templates/research_template.md` の形式に沿ってリサーチシートを作成
5. ユーザーにリサーチ結果を提示し、追加情報や修正があるか確認

### フェーズ2: 営業台本作成

リサーチ結果と自社商材情報をもとに台本を作成する。

1. ユーザーに自社の商材・サービス情報を確認（未提供の場合）
2. `templates/script_template.md` をベースに、リサーチ結果を反映した台本を作成
3. アイスブレイク → ヒアリング → 提案 → クロージングの流れで構成
4. 相手の課題仮説に基づく具体的な質問例とトーク例を記載
5. ユーザーに台本を提示し、調整があるか確認

### フェーズ3: 議事録フィードバック

商談後、ユーザーから議事録テキストまたはファイルを受け取り分析する。

1. 議事録の内容を読み込み、商談の流れを把握
2. 台本の各フェーズ（アイスブレイク、ヒアリング、提案、クロージング）に沿って評価
3. 良かった点（Good）と改善点（More）を具体的に記載
4. 次回に向けた具体的なアクションプランを3つ以上提案
5. `templates/feedback_template.md` の形式でレポートを作成し提示

## 注意事項

- フェーズ1〜3は必ずしも連続で実行する必要はない。ユーザーの依頼に応じて個別に実行可能。
- 台本はあくまでベースであり、相手の反応に応じた柔軟な対応が重要であることをユーザーに伝える。
- 議事録フィードバックはテキストベースの分析であり、声のトーンや表情などの非言語情報は含まれない旨を明記する。
- Google Driveからの議事録取得は、ユーザーにテキストの貼り付けまたはファイルの共有を依頼する形で対応する。
