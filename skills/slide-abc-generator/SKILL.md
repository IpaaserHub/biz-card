# Slide ABC Generator

画像からA（デザインシステム）・B（スライドタイプ）・C（コンテンツ原稿）を自動生成するスキル。

## 概念

| 要素 | 別名 | 役割 | 変更頻度 |
|------|------|------|----------|
| A：デザインシステム | 「憲法」 | 見た目・トーンのルールブック | 固定（ブランド統一） |
| B：スライドタイプ | 「金型」 | 構造・流れのテンプレート | 固定（論理構成） |
| C：コンテンツ原稿 | 「材料」 | 今回伝えたい具体的な中身 | 毎回変わる変数 |

**真髄：** AとBを一度定義すれば、Cを差し替えるだけで同スタイルの資料を量産できる（コンテンツとデザインの分離）。

## ワークフロー

1. 画像を視覚的に解析する（`file` ツールの `view` アクションを使用）
2. `scripts/extract_abc.py` を実行してAI解析・3ファイル生成を行う
3. 生成された3ファイルをすべて添付してユーザーに提示する

## 実行手順

### Step 1: 画像の解析

添付画像を `file` ツールの `view` アクションで確認し、以下を把握する：
- 使用色・フォント・レイアウトなどのビジュアル要素
- スライド構成・ページタイプ・情報の流れ
- 記載されているテキスト・データ・メッセージ

### Step 2: スクリプトの実行

```bash
python /home/ubuntu/skills/slide-abc-generator/scripts/extract_abc.py \
  --image <画像ファイルの絶対パス> \
  --output <出力先ディレクトリ（例：/home/ubuntu/abc_output）>
```

スクリプトは以下の3ファイルを生成する：
- `design_system_A.md` — A：デザインシステム
- `slide_types_B.md` — B：スライドタイプ
- `content_draft_C.md` — C：コンテンツ原稿

画像が複数枚ある場合は `--image` を繰り返し指定する：
```bash
python /home/ubuntu/skills/slide-abc-generator/scripts/extract_abc.py \
  --image /path/to/image1.png \
  --image /path/to/image2.png \
  --output /home/ubuntu/abc_output
```

### Step 3: 結果の提示

生成された3ファイルをすべて `message` ツールの `attachments` に含めてユーザーに提示する。各ファイルの役割と使い方を簡潔に説明する。

## 出力フォーマット

各ファイルの詳細テンプレートは `references/output_templates.md` を参照。

## 注意事項

- テキストが読み取れない場合でも、ビジュアル要素（色・レイアウト・構造）からAとBは推定できる
- Cは画像内のテキストを忠実に抽出し、読み取れない部分は `[要確認]` と記載する
- 出力ディレクトリが存在しない場合、スクリプトが自動作成する
